Core Concept

What is a Function?

As your programs grow, you will find yourself writing the same logic in multiple places. Without functions, every change to that logic requires you to find and update every copy. One missed update means a bug.

A function is a named, reusable block of code. You write the logic once, give it a name, and run it by calling that name wherever you need it. If the logic changes, you update it in one place and every call benefits automatically.

Functions are one of the most important building blocks in programming. Everything you have learned so far becomes more powerful when wrapped in functions that can be called and reused.

3
Core Concept

Defining a Function

You define a function using the function keyword, followed by a name, parentheses, and a block of code in curly braces. Ideally, the name should describe what the function does. The code inside the braces is called the function body. It does not run when you define the function but only runs when you call it.

JavaScript 
function showWelcome() {
  console.log("Welcome to the app")
  console.log("You are now logged in")
}

// Nothing prints. The function is defined but not yet called
Output

Run the code above. Nothing will be printed to the output. Defining a function stores the body but only executes when you call it by name.

4
Core Concept

Calling a Function

To run a function, write its name followed by parentheses. This is called a function call. Every time you call it, the entire function body runs from top to bottom.

JavaScript 
function showWelcome() {
  console.log("Welcome to the app")
  console.log("You are now logged in")
}

showWelcome()  // first call
showWelcome()  // second call -> same body runs again
Output

One of the main benefits of functions is that the same logic runs on every call without repeating the code. If you needed to call it fifty times, you would still write the body only once.

JavaScript 
function printDivider() {
  console.log("--------------------")
}

console.log("Monday")
printDivider()
console.log("Tuesday")
printDivider()
console.log("Wednesday")
printDivider()

Core Concept

Parameters and Arguments

The functions so far do the same thing every time. To make a function flexible, you give it parameters which are named placeholders defined in the parentheses when you write the function. When you call that function, you pass in arguments which are the actual values that fill those placeholders for that call.

Parameters are the names in the definition. Arguments are the values you pass at the call site.

In the example below, track and duration are the parameters and are defined in the function signature and act as placeholders. When you call the function, the values you provide, which are "Midnight Rain" and 214, are the arguments. They fill those parameters for that specific call. Each new call in the code below passes different arguments and gets a different result.

JavaScript 
// "track" and "duration" are parameters
function playTrack(track, duration) {
  console.log(`Now playing: ${track} (${duration}s)`)
}

// "Midnight Rain" and 214 are arguments
playTrack("Midnight Rain", 214)
playTrack("Golden Hour", 187)
playTrack("Blinding Lights", 200)
Output

Order matters

Arguments are assigned to parameters in the order they are listed. The first argument goes to the first parameter, the second to the second, and so on. If you pass them in the wrong order, the function receives the wrong values and produces a wrong result without raising an error.

JavaScript 
function describePlace(country, capital) {
  console.log(`The capital city of ${country} is ${capital}`)
}

describePlace("Kenya", "Nairobi")   // correct
describePlace("Nairobi", "Kenya")   // swapped -> reads nonsensically with no error
Output

Missing arguments

If you call a function with fewer arguments than it has parameters, the missing ones receive the value undefined. Printing the parameters directly is the clearest way to see this.

JavaScript 
function resizeImage(width, height) {
  console.log(`width: ${width}`)
  console.log(`height: ${height}`)
}

resizeImage(1920, 1080)   // both provided
console.log("---")
resizeImage(1920)          // height is missing -> will print undefined

Core Concept

Return Values

So far every function has printed directly to the console. But printing is not always the goal. Often you need a function to calculate something and hand the result back so you can use it elsewhere. This includes storing it in a variable or passing it to another function. That is what the return keyword does.

When a function hits a return statement, it immediately stops and sends the value back to wherever it was called from. Any code after return in the function body does not run.

One important thing to note: a returned value is not automatically printed. To see it, you still need to pass the function call or it's result to console.log.

JavaScript 
function add(a, b) {
  return a + b
}

// The returned value can be stored in a variable
let result = add(3, 4)
console.log(result)  // 7

// Or used directly in an expression
console.log(add(3, 4) * 2)  // 14

// Or embedded in a template string
console.log(`The sum is ${add(3, 4)}`)  // The sum is 7
Output

Here is another complete example. The function below checks stock levels and returns a status string. Notice two things. First, each return exits the function immediately so there is no need for else if. Once a condition matches and a value is returned, the function stops and the remaining if statements are never reached. This is called an early return. If all the if statements are false, then the last return statement will run.

Second, the if statements have no curly braces. When a condition and its result fit on one line, the braces are optional. It is a style choice here to keep the pattern compact and readable. The logic is simple enough that the braces would add noise rather than clarity. You don’t have to write your code this way. At this stage, it’s perfectly fine (and often better) to stick with curly braces. Just be aware that this alternative style exists, and you’ll come across it in real-world code.

JavaScript 
function stockStatus(units) {
  if (units === 0)  return "Out of stock"
  if (units < 10)  return "Low stock"
  if (units < 50)  return "In stock"
  return "Well stocked"
}

console.log(stockStatus(0))    // Out of stock
console.log(stockStatus(5))    // Low stock
console.log(stockStatus(30))   // In stock
console.log(stockStatus(200))  // Well stocked
Output

Core Concept

Scope

Scope refers to where a variable is accessible in your code. A variable declared with let inside a function only exists within that function. It is created when the function runs and becomes inaccessible when it finishes executing. Code outside the function cannot see it. This is called function scope.

JavaScript 
function processOrder() {
  let fee = 4.99  // local variable -> only exists inside processOrder function
  console.log(`Processing fee: ${fee}`)
}

processOrder()

// This throws a ReferenceError because fee does not exist out here
console.log(fee)
Output

The fix is to return the value instead of trying to reach inside the function from outside. That is the designed way to get data out of a function.

JavaScript 
function processOrder() {
  let fee = 4.99
  return fee  // send the value out
}

let orderFee = processOrder()  // catch it here
console.log(`Fee: ${orderFee}`)
Output

JavaScript actually has three types of scope: function scope (what we covered above), block scope (variables declared inside an if block or loop are not accessible outside that block), and global scope (variables declared outside any function or block are accessible everywhere). The behaviour of let, const, and var differs across these, but that is beyond this course. For now, if you need a value that a function uses, simply return it.

Core Concept

Function Expressions

There is a second way to create a function. Instead of starting a statement with the function keyword, you assign the function to a variable. This is called a function expression. You call the function the same way by the writing variable name followed by parentheses.

JavaScript 
// Function declaration
function square(n) {
  return n * n
}

// Function expression -> same behaviour, different syntax
let square2 = function(n) {
  return n * n
}

console.log(square(5))   // 25
console.log(square2(5))  // 25

Summary

Module Summary

Key takeaways

  • A function is a named, reusable block of code. Define it once with the function keyword. Call it by name with parentheses whenever you need to run it.
  • Parameters are the placeholder names in the function definition. Arguments are the actual values passed when calling. Order matters. Arguments are assigned to parameters left to right.
  • If you call a function with fewer arguments than parameters, the missing ones are undefined.
  • The return keyword sends a value back to the caller and stops the function. All code after return does not run.
  • The early return pattern uses multiple return statements to exit as soon as a condition matches, avoiding unnecessary else if chains.
  • Variables declared with let inside a function are local. They only exist inside that function. Use return to pass values out.
  • A function expression assigns a function to a variable. You call it the same way by writing the variable name followed by parentheses.
