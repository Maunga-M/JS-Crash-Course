Core Concept

Arithmetic Operators

Now that you can store values in variables, you need a way to do something with them. Arithmetic operators let you perform calculations on numbers. There are six in JavaScript. Each one takes two values, called operands, and produces a result.

Addition

Operator: +

Adds two numbers together.

JavaScript 
console.log(120 + 35)   // 155
console.log(4.5 + 1.5)  // 6
Output

Subtraction

Operator: -

Subtracts the second number from the first.

JavaScript 
console.log(200 - 45)   // 155
console.log(10 - 14.5)  // -4.5
Output

Multiplication

Operator: *

Multiplies two numbers.

JavaScript 
console.log(12 * 9)    // 108
console.log(2.5 * 4)  // 10
Output

Division

Operator: /

Divides the first number by the second.

Note: If you divide any number by zero, JavaScript does not throw an error. It returns a special value called Infinity. This is JavaScript's way of representing a number too large to express. You will see it appear in real code whenever a division by zero sneaks in unnoticed.

JavaScript 
console.log(99 / 3)   // 33
console.log(7 / 2)    // 3.5
console.log(5 / 0)    // Infinity
Output

Modulo

Operator: %

Returns the remainder after dividing two numbers as evenly as possible. For example, 13 divided by 5 goes in two times with 2 left over, so 13 % 5 gives you 2. A common use is checking whether a number is even i.e if dividing it by 2 leaves no remainder, then it is even.

JavaScript 
console.log(10 % 3)  // 1
console.log(8 % 2)   // 0 -- 8 is even
Output

Exponentiation

Operator: **

Raises a number to a power. E.g 4 ** 5 means 4 multiplied by itself 5 times.

JavaScript 
console.log(2 ** 8)   // 256
console.log(3 ** 3)   // 27
Output

Shortcuts

A common task in programming is updating a variable by performing an operation on its current value. You could write x = x + 5, but JavaScript provides shorter ways to express the same thing. These are called assignment operators.

Shorthand Assignment Operators

The operators you have used in the previous section can be used in shorthand assignment

+= → addition assignment

-= → subtraction assignment

*= → multiplication assignment

/= → division assignment

%= → modulo (remainder) assignment

**= → exponentiation assignment

Each of these combines an arithmetic operation with assignment. The operator reads: take the current value, apply the operation, store the result back in the same variable.

JavaScript 
let points = 100

points += 20   // same as: points = points + 20
console.log(points)  // 120

points -= 15   // same as: points = points - 15
console.log(points)  // 105

points *= 2    // same as: points = points * 2
console.log(points)  // 210

points /= 3    // same as: points = points / 3
console.log(points)  // 70

points %= 6    // same as: points = points % 6
console.log(points)  // 4

points **= 2   // same as: points = points ** 2
console.log(points)  // 16
Output

Increment and Decrement Operators

The increment operator ++ adds 1 to a variable. The decrement operator -- subtracts 1. These are most commonly used inside loops where you need to count up or down one step at a time.

JavaScript 
let count = 0

count++
console.log(count)  // 1

count++
console.log(count)  // 2

count--
console.log(count)  // 1
Output

Core Concept

Order of Operations

When an expression contains more than one operator, JavaScript does not simply evaluate from left to right. It follows a fixed order of precedence. Knowing this order is important because getting it wrong produces an incorrect result with no error message to warn you.

The order is as follows. Expressions inside parentheses are evaluated first. Then exponentiation. Then multiplication, division, and remainder, evaluated left to right. Finally addition and subtraction, also left to right.

JavaScript 
// Multiplication happens before addition
console.log(2 + 3 * 4)    // 14

// Parentheses force a different order
console.log((2 + 3) * 4)  // 20

// Exponentiation before multiplication
console.log(2 * 3 ** 2)   // 18

// Parentheses to control exponentiation
console.log((2 * 3) ** 2) // 36
Output
A silent mistake

Order of operations errors do not produce any error message. The code runs, produces a wrong number, and moves on. When a calculation gives you an unexpected result, check whether parentheses are in the right place.

Summary

Module Summary

Key takeaways

  • JavaScript has six arithmetic operators: +, -, *, /, %, and **.
  • The remainder operator % returns what is left over after division, not the quotient.
  • Shorthand operators like += and /= update a variable in place.
  • The increment operator++ adds 1 to a variable. The decrement operator-- subtracts 1.
  • The order of operations affects the result. Parentheses come first. Then exponentiation. Multiplication, division, and remainder are evaluated before addition and subtraction.
