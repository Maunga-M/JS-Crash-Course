Core Concept

What is an Array?

So far every variable you have declared holds one value. That works for one price or one name. But what if you need to work with a week of temperature readings or a list of cities or fifty product prices? Declaring a separate variable for each one is not practical. An array solves this problem by letting you store multiple values in a single variable, in a specific order.

An array is an ordered collection of values. Each value sits at a numbered position called an index, starting from 0. You can put any type of value into an array, even other arrays (more on this later). The array keeps them in the order you put them and lets you access any one by its position.

Core Concept

Creating an Array

The standard way to create an array is with square brackets. If elements are already present, they should be separated by commas.

JavaScript 
// Array with values
let oceans = ["Pacific", "Atlantic", "Indian"]

// Empty array -> values added later
let results = []

// Mixed types -> valid but not recommended
let mixed = ["Nairobi", 1963, true]

console.log(oceans)
console.log(results)
console.log(mixed)
Output
Use descriptive names

Name your array after what it contains. fruits, scores, and bookings tell you immediately what is in the array. Names like list, arr, or data do not. The more specific the name, the easier your code is to read and debug.

Keep arrays to one type

JavaScript allows you to mix types in the same array, but in practice you almost never should. An array of strings and an array of numbers are both predictable because you know what you are working with on every iteration. A mixed array forces you to check the type of each element before using it, which adds complexity and makes bugs harder to spot. Treat it as a strong convention: one type per array.

JavaScript 
let planets   = ["Mercury", "Venus", "Earth", "Mars"]
let distances  = [77, 261, 78, 225]  // millions of km from Earth
let inOrbit    = [true, true, false, true]

console.log(planets)
console.log(distances)
console.log(inOrbit)
Output

Core Concept

Accessing Elements

You access an element by writing the array name followed by the index in square brackets. Indexing starts at 0, so the first element is at index 0, the second at index 1, and so on. You saw this same pattern with strings in Module 3. Arrays work the same way.

JavaScript 
let mountains = ["Everest", "K2", "Kangchenjunga", "Lhotse"]

console.log(mountains[0])   // Everest -> first element
console.log(mountains[2])   // Kangchenjunga -> third element
Output

Like strings, we can also access the length property to know how many elements are in the array. Because indexing starts at 0, the last element always sits at index length - 1. If you try to access an index that does not exist, JavaScript does not throw an error. It returns undefined.

JavaScript 
let mountains = ["Everest", "K2", "Kangchenjunga", "Lhotse"]

console.log(mountains.length)                  // 4
console.log(mountains[mountains.length - 1])   // Lhotse -> last element
console.log(mountains[10])                    // undefined -> no element at index 10
Output

You can also update an element by assigning a new value to its index directly.

JavaScript 
let rivers = ["Nile", "Amazon", "Congo"]

rivers[1] = "Yangtze"
console.log(rivers)  // ["Nile", "Yangtze", "Congo"]
Output

Mutating an Array

We can use in-built methods to help us make changes to an array or extract elements from it. JavaScript has many array methods. Here are a few you should know.

push()

Adds one or more elements to the end of the array. Returns the new length.

JavaScript 
let queue = ["Alice", "Bob"]
queue.push("Carol")
console.log(queue)  // ["Alice", "Bob", "Carol"]

// You can push more than one element at a time
queue.push("Dave", "Eve")
console.log(queue)  // ["Alice", "Bob", "Carol", "Dave", "Eve"]
Output

pop()

Removes the last element from the array and returns it. The original array is shortened by one.

JavaScript 
let queue = ["Alice", "Bob", "Carol"]
let removed = queue.pop()
console.log(removed)  // Carol
console.log(queue)    // ["Alice", "Bob"]
Output

unshift()

Adds one or more elements to the beginning of the array. All existing elements shift right to make room.

JavaScript 
let playlist = ["Song B", "Song C"]
playlist.unshift("Song A")
console.log(playlist)  // ["Song A", "Song B", "Song C"]
Output

shift()

Removes the first element from the array and returns it. All remaining elements shift left by one position.

JavaScript 
let playlist = ["Song A", "Song B", "Song C"]
let first = playlist.shift()
console.log(first)     // Song A
console.log(playlist)  // ["Song B", "Song C"]
Output

slice()

Extracts a portion of the array and returns it as a new array. It does not change the original. It takes a start index and an optional end index which is not included in the extraction. If an end index is not given, elements after the start index are extracted. You saw the same method on strings in Module 3.

JavaScript 
let medals = ["Gold", "Silver", "Bronze", "Finalist", "Participant"]

console.log(medals.slice(0, 3))   // ["Gold", "Silver", "Bronze"]
console.log(medals.slice(2))       // ["Bronze", "Finalist", "Participant"]
console.log(medals)                 // original unchanged
Output
There are many more methods

JavaScript arrays have dozens of built-in methods such as splice, find, filter, map, reduce, sort etc. The ones covered here are the foundations. You will encounter the rest as you build more complex programs.

Copying an Array

Recall from Module 1 that JavaScript has two groups of types: primitives (strings, numbers, booleans) and objects (arrays, objects, functions). This distinction matters when copying a value into another variable. If you assign a primitive to a new variable and then change the original, the copy is unaffected because each variable holds its own value in the computer's memory.

JavaScript 
// Strings are copied by value -> each variable is independent
let city  = "Lagos"
let city2 = city
city2 = "Accra"

console.log(city)   // Lagos -> unchanged
console.log(city2)  // Accra
Output

Arrays behave differently. When you assign an array to a new variable, both variables point to the same array in memory. This is called copying by reference. There is no second array. Just two variable names pointing to the same array. If you change the array through either variable, the other one reflects the change immediately.

JavaScript 
// Arrays are copied by reference and both variables point to the same array
let original = [10, 20, 30]
let copy     = original

copy.push(40)

console.log(original)  // [10, 20, 30, 40] -> also changed
console.log(copy)      // [10, 20, 30, 40]
Output

Making a true copy

To create an independent copy of an array, you need to explicitly produce a new one. One reliable way is to call slice() with no arguments. This returns a new array containing all the same elements, completely separate from the original.

JavaScript 
let original = [10, 20, 30]
let copy = original.slice()

copy.push(40)

console.log(original)  // [10, 20, 30] -> untouched
console.log(copy)      // [10, 20, 30, 40]
Output

Searching an Array

There are a few methods we can use to find a specific element or it's index

includes()

Returns true if the value is found in the array, false if it is not. Use this when you only need to know whether an element exists in the array.

JavaScript 
let languages = ["Python", "JavaScript", "Rust"]

console.log(languages.includes("JavaScript"))  // true
console.log(languages.includes("Go"))           // false
Output

indexOf()

Returns the index of the first occurrence of the value. If the value is not found, it returns -1. Use this when you need to know where in the array something is, not just whether it exists.

JavaScript 
let languages = ["Python", "JavaScript", "Rust"]

console.log(languages.indexOf("Python"))      // 0
console.log(languages.indexOf("JavaScript"))  // 1
console.log(languages.indexOf("Go"))          // -1 -> not found
Output

There is a trap with indexOf that you must watch out life. Because indexOf returns 0 when the item is at the first position, and 0 is falsy, using indexOf directly inside an if condition produces the wrong result for the first element. The code below shows the problem and two ways to fix it.

JavaScript 
let fruits = ["apple", "banana", "mango"]

// WRONG -> indexOf("apple") returns 0, which is falsy
// This block will NOT run even though "apple" is in the array
if (fruits.indexOf("apple")) {
  console.log("This will not run")  // never runs
}

// CORRECT -> compare against -1 explicitly
if (fruits.indexOf("apple") !== -1) {
  console.log("This will run")
}

// SIMPLEST -> use includes() when you only need to know if it exists
if (fruits.includes("apple")) {
  console.log("This will also run")
}
Output
Use includes() for existence checks

If you only need to know whether something is in the array, use includes(). Reserve indexOf() for when you actually need the position. This avoids the trap entirely.

Core Concept

Multi-Dimensional Arrays

An array can contain other arrays as its elements. When each element is itself an array, the result is called a two-dimensional array. This is useful for representing data that has rows and columns such as a table of records, a grid, or any structure where each entry has multiple fields.

To access an element inside a nested array, you chain two sets of square brackets. The first bracket selects which inner array you want. This gives you an entire row. The second bracket selects a specific element within that inner array which then gives you the value at that column. So fruits[1][2] means: go to the element at index 1 of the outer array, then within that inner array, go to the element at index 2.

JavaScript 
// Each inner array is one country's data: [country, capital, population]
let countries = [
  ["Nigeria",   "Abuja",       220000000],
  ["Kenya",     "Nairobi",     55000000],
  ["Ethiopia",  "Addis Ababa", 128000000]
]

// countries[1] → the Kenya row → ["Kenya", "Nairobi", 55000000]
// countries[1][1] → index 1 within that row → "Nairobi"
console.log(countries[1][1])   // Nairobi

// countries[2] → the Ethiopia row
// countries[2][2] → index 2 within that row → 128000000
console.log(countries[2][2])   // 128000000

// You can also grab an entire row
console.log(countries[0])       // ["Nigeria", "Abuja", 220000000]
Output

Summary

Module Summary

Key takeaways

  • An array stores multiple values in order. Index starts at 0. Access with arr[i]. Out-of-range access returns undefined.
  • Keep arrays to one type as a best practice. Mixed-type arrays are valid but harder to work with reliably.
  • push() and pop() add and remove from the end. unshift() and shift() add and remove from the beginning. slice() extracts without changing the original.
  • Assigning an array to a new variable does not create a copy because both point to the same array. Use slice() to make an independent copy.
  • includes() returns true or false. indexOf() returns a position or -1.
  • A 2D array is an array of arrays. arr[row][col] first selects the inner array at that row, then selects the element at that column within it.
