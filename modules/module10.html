Core Concept

What is an Object?

Arrays are great for ordered lists of similar values such as a list of scores or names. But sometimes you need to describe a single thing that has multiple distinct properties. For example, a book has a title, an author, a year, and a page count. A user has a name, an email address, and an account status. These are not lists of similar items but collections of different pieces of information that belong to one entity.

An object lets you group related data under named keys. It is not indexed. So instead of accessing values by their position like in an array or string, you access them by name.

Use an array when order matters and the values are of the same kind. Use an object when you are describing a single entity with named properties.

3
Core Concept

Syntax

You create an object with curly braces. Inside, each entry is a key-value pair separated by a colon. Multiple pairs are separated by commas. The keys are called properties. The values can be any type be it strings, numbers, booleans, arrays, or even other objects.

By convention, keys are written without quotes when they follow standard variable naming rules. You can also write them with quotes . Both forms work, but unquoted is more common. However, you must use quotes if the key contains a space or special character.

JavaScript 
let satellite = {
  name:        "Hubble",
  launchYear:  1990,
  operational: true,
  orbitKm:     547
}

// Keys can also be written with quotes -> Works just fine
let satellite2 = {
  "name":        "James Webb",
  "launchYear":  2021,
  "operational": true,
  "orbitKm":     1500000
}

console.log(satellite)
console.log(satellite2)
Output
Objects, arrays, and functions are all objects

In JavaScript, object is the umbrella type for anything that is not a primitive. An object literal, an array, and a function are all objects under the hood. They all have properties and methods. This is why typeof {} and typeof [] both return "object". It also explains why arrays have built-in properties like length and methods like push(). To check specifically whether something is an array, use Array.isArray() rather than typeof.

Core Concept

Accessing Properties

Dot notation

The most common way to access a property is to use dot notation by writing the object name, a dot and then the property name. Use this whenever you know the property name at the time you are writing the code.

JavaScript 
let planet = {
  name:     "Mars",
  moons:    2,
  diameter: 6779,
  hasWater: true
}

console.log(planet.name)
console.log(planet.moons)
console.log(`${planet.name} has ${planet.moons} moon(s)`)
Output

Bracket notation

Bracket notation lets you access a property using a string inside square brackets. You’ll need it in two main cases: when the property name doesn’t follow normal variable naming rules (so dot notation won’t work), and when the property name is stored in a variable.

JavaScript 
let profile = {
  "full name":    "Mwangi Cheza",
  "sign-up year": 2022,
  "account type": "premium"
}

// dot notation would be a syntax error here
console.log(profile["full name"])
console.log(profile["sign-up year"])
console.log(profile["account type"])
Output

When the property name is stored in a variable, bracket notation accepts the variable directly. Dot notation only works with a name written literally in the code.

JavaScript 
let settings = {
  theme:    "dark",
  language: "en",
  fontSize: 16
}

let field = "theme"
console.log(settings[field])  // same as settings["theme"]

console.log(settings.field)  // doesn't work -> dot notation produces undefined
Output
When to use each

Use dot notation by default. It is cleaner and more readable. Use bracket notation when the key is stored in a variable, or when the key contains spaces or special characters.

Core Concept

Shorthand Properties

When you create an object and the value you want to assign to a key is already stored in a variable with the same name, you can use shorthand property syntax. Instead of writing key: variable, you write just the name once. JavaScript understands that the key and the value are the same name.

JavaScript 
let username = "kofi92"
let email    = "kofi@example.com"
let age      = 30

// Without shorthand
let user1 = {
  username: username,
  email:    email,
  age:      age
}

// With shorthand -> same result, less repetition
let user2 = { username, email, age }

console.log(user1)
console.log(user2)
Output

This is common in modern JavaScript, especially when building objects from variables you have already declared. You will see it frequently in real codebases.

Core Concept

Updating and Adding Properties

Updating a property

You update an existing property by assigning a new value to it, exactly like reassigning a variable. The key stays the same and only the value changes.

JavaScript 
let device = {
  brand: "Lenovo",
  ramGB: 16
}

device.ramGB = 32
console.log(device)
Output

Adding a property

If you assign to a key that does not exist yet, JavaScript creates it. There is no separate method for adding a new property. The same assignment syntax handles both updating and adding.

JavaScript 
let device = {
  brand: "Lenovo",
  ramGB: 16
}

device.storageGB = 512  // new key-value pair gets created
device.colour    = "black"
console.log(device)
Output

Deleting a property

To remove a property entirely, use the delete operator. After deletion, accessing that key returns undefined.

JavaScript 
let device = {
  brand:   "Lenovo",
  ramGB:   16,
  model:   "X1 Carbon"
}

delete device.model
console.log(device)
console.log(device.model)  // undefined -> key no longer exists
Output

Core Concept

Checking Existence

Sometimes you need to know whether a property exists on an object before using it. There are three approaches we can use, each with different behaviour. Understanding the difference matters because choosing the wrong one can introduce silent bugs.

if (obj.key)

The most intuitive approach is to check the property directly in an if statement. The problem is that this checks whether the value is truthy, not whether the key exists. If the key is present but its value is 0, false, or an empty string, the check fails even though the key is there.

JavaScript 
let server = {
  host:   "localhost",
  port:   0,
  secure: false
}

// PROBLEM -> port is 0 and secure is false, both falsy
// These fail even though the keys exist
if (server.port) console.log("port exists")    // does not run
if (server.secure) console.log("secure exists")  // does not run
Output

if (obj.key !== undefined)

A step up is comparing the value to undefined. A missing property returns undefined when accessed, so this catches most cases. However it still fails in one edge case: if the key exists and its value has been explicitly set to undefined, the check will incorrectly treat it as missing. This is because conditional statements only execute when the condition is truthy.

JavaScript 
let server = {
  host: "localhost",
  port: 0,
  secure: false
}

// Works for falsy values -> port and secure are now found correctly
if (server.port !== undefined) console.log("port exists")
if (server.secure !== undefined) console.log("secure exists")
if (server.timeout !== undefined) console.log("timeout exists")  // missing -- does not run
Output

"key" in obj

The in operator is the most reliable way to check existence. It checks whether the key is present on the object, regardless of what the value is including if the value is undefined, 0, false, or anything else falsy.

JavaScript 
let server = {
  host: "localhost",
  port: 0,
  secure: false
}

if ("port" in server) console.log("port exists")
if ("secure" in server) console.log("secure exists")
if ("timeout" in server) console.log("timeout exists")  // key not present -> does not run
Output

Core Concept

Looping Through Objects

Objects are not iterable the same way arrays are, so you cannot use for...of directly on an object. Instead, use for...in, which loops over the keys of an object. On each iteration you get one key as a string (there is no numeric index like in a for loop). Use bracket notation inside the loop to access the corresponding value.

JavaScript 
let stadium = {
  name:     "Wembley",
  country:  "England",
  capacity: 90000,
  opened:   2007
}

for (let key in stadium) {
  console.log(`${key}: ${stadium[key]}`)
}
Output

Core Concept

Object Methods

JavaScript provides built-in methods that extract the keys or values of an object as an array. Once you have an array, you can use all the array tools you already know i.e length, for...of, and so on.

Object.keys()

Returns an array of all the property names in the object. Saving it to a variable lets you use it in multiple places without calling the method twice.

JavaScript 
let recipe = {
  name:     "Jollof Rice",
  servings: 4,
  prepMins: 20,
  cookMins: 45
}

let keys = Object.keys(recipe)
console.log(keys)
console.log(`This recipe has ${keys.length} properties`)

for (let key of keys) {
  console.log(key)
}
Output

Object.values()

Returns an array of all the property values in the same order as the keys. Again, saving to a variable before using it keeps the code clean.

JavaScript 
let scores = {
  maths:   88,
  english: 74,
  science: 91
}

let values = Object.values(scores)
console.log(values)

for (let score of values) {
  console.log(score)
}
Output

Core Concept

Functions as Values

Because functions are objects in JavaScript, they can be stored as property values just like a string or number. When a function is stored inside an object, it is called a method. You call it using dot notation followed by parentheses.

A function stored in an object can reference other properties on the same object by using the object's variable name. This works because the object is declared in the outer scope so the function can see it the same way any code in that scope can. The issue of scope is a bit of an advanced topic so don't worry too much about it.

JavaScript 
let timer = {
  duration: 30,
  unit:     "seconds",
  start: function() {
    console.log("Timer started")
  },
  describe: function() {
    // timer is in the outer scope so this function can read its properties
    console.log(`Running for ${timer.duration} ${timer.unit}`)
  }
}

timer.start()
timer.describe()
Output

You have already been using methods throughout this course. console.log(), array.push(), and string.toUpperCase() are all functions stored as properties on objects and called using dot notation.

Arrays of Objects

One of the most common data structures you will encounter is an array of objects. Often each object has the same keys, but in practice some keys may be absent on certain objects. You access a single object by its index, then access its properties using dot notation.

JavaScript 
let runners = [
  { name: "Aisha",  timeMin: 42 },
  { name: "Carlos", timeMin: 38 },
  { name: "Priya",  timeMin: 45 }
]

// Access a single object by index, then its property
console.log(runners[0])    // Gives the first object
console.log(runners[0].name)    // Aisha
console.log(runners[1].timeMin)  // 38
Output

To process every object in the array, we can combine a loop with property access. On each iteration the loop variable holds the entire object, and you use dot notation to read from it.

JavaScript 
let runners = [
  { name: "Aisha",  timeMin: 42 },
  { name: "Carlos", timeMin: 38 },
  { name: "Priya",  timeMin: 45 }
]

for (let runner of runners) {
  console.log(`${runner.name} finished in ${runner.timeMin} minutes`)
}
Output

Core Concept

Nested Objects

A property value can itself be an object. This is called a nested object. You access its properties by chaining the obejct with first the outer key, then the inner key. We will only go one level deep here, but as you advances you will encounter nested objects that are even higher than four levels.

JavaScript 
let company = {
  name:    "Savanna Tech",
  founded: 2018,
  address: {
    city:    "Kampala",
    country: "Uganda"
  }
}

console.log(company.name)
console.log(company.address.city)
console.log(company.address.country)
console.log(`${company.name} is based in ${company.address.city}, ${company.address.country}`)
Output

Module Summary

Key takeaways

  • An object stores related data as named key-value pairs. Use objects when data has named properties; use arrays when order matters and values are of the same kind.
  • Keys can be written with or without quotes. Use quotes when the key contains a space or special character.
  • Access properties with dot notation by default. Use bracket notation when the key is in a variable or contains spaces or special characters.
  • Shorthand property syntax lets you write { name } instead of { name: name } when the variable and key share the same name.
  • Assign to a key to update or add it. Use delete obj.key to remove a property entirely.
  • Use "key" in obj to check existence reliably. Avoid if (obj.key) for falsy values.
  • for...in loops over keys with no numeric index. Object.keys() and Object.values() return arrays you can loop with for...of.
  • A function stored as a property is called a method. It can reference other properties on the same object by using the object's name.
  • In arrays of objects, often each element shares the same keys, but some keys may be absent on certain objects.
  • Nested objects store an object as a property value. Access inner properties by chaining dot notation: obj.outer.inner.
