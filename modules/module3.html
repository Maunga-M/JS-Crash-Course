Core Concept

Quote Rules

In Module 1 you learned that a string is text wrapped in quotes. This also includes:

  • empty quotes - ""
  • quotes with a single space - " "
  • quotes with 10 characters - 'I am happy'
  • quotees with special characters - '@!!!!'

All of these are strings. JavaScript allows you to create your string using single quotes and double quotes. Both work exactly the same way. The only rule is that you must open and close with the same type.

JavaScript 
console.log("Lake Victoria")   // double quotes
console.log('Lake Victoria')   // single quotes -- same result
Output

The practical reason to choose one over the other is when your string itself contains a quote character. If your text includes a single quote, wrap the string in double quotes so JavaScript does not get confused about where the string ends. The same logic applies in reverse.

JavaScript 
// String contains a single quote -- use double quotes to wrap it
console.log("It's a sunny day")

// String contains a double quote -- use single quotes to wrap it
console.log('She said "hello"')
Output

But what if you genuinely need both types of quote inside the same string? For example: She said, "I'll be there soon." Neither single nor double quotes can wrap that cleanly. The solution is the escape character \. Placing a backslash before a quote inside the string tells JavaScript to treat it as a literal character and not as the end of the string.

JavaScript 
// Escape the inner quotes with a backslash
console.log("She said, \"I\'ll be there soon.\"")

console.log('It\'s the \"best\" day')
Output
Mismatched quotes

Opening with one type and closing with another causes a syntax error. JavaScript reads the first quote as the start of the string and looks for the matching closing quote. If it finds a different one first, it gets confused and throws an error before your code even runs.

Core Concept

String Operations

Strings are not numbers, so most arithmetic operations does not make sense on them. There is one operation that works on strings: addition. When you use + between two strings, it joins them end to end. This is called concatenation.

Every other arithmetic operator, does not work on strings. When JavaScript cannot produce a meaningful number from an operation, it returns a special value called NaN, which stands for Not a Number. It is JavaScript's way of saying the this calculation does not make sense.

JavaScript 
console.log("hello" - "world")  // NaN
console.log("hello" * 2)       // NaN
console.log("hello" / 2)       // NaN
Output
NaN is still a number type

Even though NaN means Not a Number, typeof NaN returns "number". This is a well-known quirk of JavaScript. Think of it as a number-shaped hole where a valid result should be. You cannot do further arithmetic with it either. Any operation involving NaN returns NaN.

Core Concept

Concatenation

As defined in the previous section, concatenation means joining strings together using the + operator. The strings are glued end to end exactly as written, so if you need a space between words you have to include it yourself eiher.

JavaScript 
// Without a space -- the words run together
console.log("Indian" + "Ocean")

// With a space included
console.log("Indian" + " " + "Ocean")

// Concatenating more than two strings
console.log("The " +"Pacific" + " " + "Ocean" + " is the largest")
Output

Note how we can at the end at the start or end of the string, or as its own string

Concatenation also works with variables. The variable is replaced by its value before the strings are joined.

JavaScript 
let ocean  = "Atlantic"
let suffix = "Ocean"

console.log(ocean + " " + suffix)
console.log("The " + ocean + " " + suffix + " borders five continents")
Output

Concatenation gets messy quickly when you have several variables. A better approach is to use template strings.

Core Concept

Template Strings

A template string is the third way to write a string in JavaScript, using backtick characters instead of quotes. The backtick key sits in the top-left of most keyboards, below the Escape key. Template strings have two advantages over regular strings. They can span multiple lines, and they let you embed variables and expressions directly inside the text using ${}.

Basic template string

A template string can be assigned to a variable just like any other string. The multi-line version preserves the line breaks exactly as you write them, which regular strings cannot do. We can even put quotes inside the template string to display them.

JavaScript 
let greeting = `"Hey there!"`
console.log(greeting)

let poem = `Roses are red
Violets are blue
JavaScript is fun`
console.log(poem)
Output

To see why this matters, here is what happens when you try to do the same thing with a regular string. Entering the next line inside a regular string causes a syntax error because JavaScript expects the closing quote on the same line.

JavaScript 
// This throws a SyntaxError -- regular strings cannot span lines
let broken = "Roses are red
Violets are blue"
Output

Embedding variables and expressions

If you place a variable or any expression inside ${}, it is evaluated and inserted into the string. This is far cleaner than concatenation when you have more than one value to include.

JavaScript 
let mountain = "Kilimanjaro"
let height   = 5895

// With concatenation
console.log("Mount " + mountain + " is " + height + " metres tall")

// With a template string. Same result but much easier to read
console.log(`Mount ${mountain} is ${height} metres tall`)

// You can even put expressions inside ${ }
console.log(`Height in feet: ${height * 3.281}`)
Output

Indexing and Length

A string is a sequence of characters, and each character sits at a numbered position called an index. This string indexing starts at 0. The first character is at index 0, the second at index 1, and so on. You access a specific character by writing the string (or variable) followed by the index in square brackets. Trying to access an index that is out of range will result in undefined.

JavaScript 
let river = "Nile"

console.log(river[0])  // N
console.log(river[1])  // i
console.log(river[3])  // e
console.log(river[5])  // undefined - out of range!
Output

The Length Property

The length property tells you how many characters are in a string, including spaces and punctuation. We can accessing this property by putting .length after the string or variable name. Because indexing starts at 0, the last character always sits at index length - 1.

JavaScript 
let river = "Amazon"

console.log(river.length)              // 6
console.log(river[river.length - 1])  // n (last character)
Output

Core Concept

String Methods

Strings come with built-in methods that let you inspect and transform text. Methods are fucntions that belong to objectis. In JavaScript, even though a string is a primitive type, the language temporarily wraps it in a special String object behind the scenes whenever you call a method on it. This mechanism is beyond the scope of this scope but just know that it happens automatically. You call a method by writing the variable name, a dot, and the method name with parentheses. JavaScript has many string methods. Here are some of them.

toUpperCase() and toLowerCase()

Convert all characters to upper or lower case. The original string is not changed. A new string is returned.

JavaScript 
let country = "South Africa"

console.log(country.toUpperCase())  // SOUTH AFRICA
console.log(country.toLowerCase())  // south africa
console.log(country)  // South Africa - not changed!
Output

trim()

Removes whitespace from both ends of a string. Useful when handling user input that may have accidental spaces before or after the text.

JavaScript 
let input = "  Tanzania  "

console.log(input.length) // 12 - spaces counted
console.log(input.trim()) // "Tanzania"
console.log(input.trim().length) // 8
Output

includes()

Returns true if the string contains the specified text, and false if it does not. It is case-sensitive.

JavaScript 
let sentence = "The Congo river runs through the rainforest"

console.log(sentence.includes("Congo")) // true
console.log(sentence.includes("Nile")) // false
console.log(sentence.includes("congo")) // false -- case-sensitive
Output

indexOf()

Returns the index of the first occurrence of the specified text. If not found, it returns -1. We can also find the last occurence using the method lastIndexOf()

JavaScript 
let continent = "South America and North America"

console.log(continent.indexOf("America")) // 6
console.log(continent.lastIndexOf("America")) // 24 - the last occurence
console.log(continent.indexOf("Africa")) // -1
Output

slice()

Extracts a portion of a string and returns it as a new string. It takes two arguments: the index to start from, and the index to stop before. The character at the stop index is not included. If there is no stop index, then the extract goes all the way to the end of the string

JavaScript 
let place = "Sahara Desert"

console.log(place.slice(0, 6)) // Sahara (space character at index 6 excluded)
console.log(place.slice(7)) // Desert (no end index means go to the end)
console.log(place.slice(-4)) // sert (negative index. No stop index)
console.log(place.slice(-9, -1)) // Sahara (Negative index. 't' character at index -1 excluded)
Output
Negative indexes

We can use negative indices when slicing. When counting from the start, the first character is at index 0. But when counting from the end, the last character is at index -1, the second to last is -2, and so on. The reason negative counting starts at -1 and not 0 is that -0 and 0 are the same number. There can only be one zero, so the end starts at -1 instead.

The negative index only changes where slice starts. It does not change the direction. Slice always reads left to right from that starting point to the end of the string. So slice(-3) on the string "Planet" starts at the "n" which is three characters from the end, then reads forward giving you "net", not "Plan".

⚠️In JavaScript, negative indexes only work when slicing. Using a negative index without slicing results in undefined

JavaScript 
let planet = "Jupiter"

// Positive index - works
console.log(planet[0])   // J

// Negative index - does not work on strings directly
console.log(planet[-1])  // undefined
Output

endsWith()

Returns true if the string ends with the specified text. Like includes(), it is case-sensitive.

JavaScript 
let filename = "report.pdf"

console.log(filename.endsWith(".pdf"))   // true
console.log(filename.endsWith(".csv"))   // false
Output
These are not all the methods

JavaScript has many more string methods. startsWith(), replace(), split(), repeat(), and others are all available.

Module Summary

Key takeaways

  • Strings can use single or double quotes. Choose based on whether your text contains either quote character. Open and close with the same type.
  • Strings support concatenation with +. Other operations between strings return NaN.
  • NaN means Not a Number. Its type is still "number".
  • Template strings use backticks. They support multi-line text and embed values with ${}.
  • String characters are indexed from 0. Access a character with str[i], where i is the index. The length is given by str.length.
  • We can useful methods like slice() to transform strings.
