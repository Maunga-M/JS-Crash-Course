Core Concept

What are Loops?

In the previous module you worked with arrays that hold multiple values. But having an array of fifty scores does not help much if you have to write fifty separate lines of code to process each one. A loop solves this by letting you write the logic once and repeat it automatically for as many iterations as you need.

A loop runs a block of code repeatedly. On each iteration it can work with a different value, update a counter, check a condition, or accumulate a result. When the loop finishes, all that work is done. Without loops, any task that involves a collection of values would require you to write the same code over and over for each element element, with no way to handle collections of unknown size.

JavaScript has several ways to write loops. In this module we will look at three of them. Each one is suited to a different situation, and knowing which to reach for is as important as knowing how to write them.

Core Concept

for Loops

Use a for loop when you know exactly how many times you need to iterate before the loop starts. It has three parts in its parentheses, separated by semicolons.

JavaScript 
// for (initialization; condition; update)
for (let i = 0; i < 5; i++) {
  console.log(i)
}
Output

Initialization - let i = 0 creates the counter and sets its starting value.
Condition - i < 5 is checked before every iteration. When it becomes false the loop stops.
Update - i++ runs after each iteration, advancing the counter. In this example the loop runs five times prints 0, 1, 2, 3, 4 and stops before i reaches 5.

Where the for loop becomes especially useful is when working with arrays. Instead of a fixed number like 5, the condition uses arr.length, so the loop runs exactly as many times as there are elements, no matter how long the array is. The index variable i then gives you access to each element in turn.

JavaScript 
let speeds = [120, 85, 140, 95, 110]

// i starts at 0, runs while i is less than the array length, i increments by 1 each time
for (let i = 0; i < speeds.length; i++) {
  console.log(`Speed ${i + 1}: ${speeds[i]} km/h`)
}
Output

The index variable gives you precise control and you can print position numbers, access elements from two arrays at the same position, or loop backwards. The same structure also works on strings. Because strings are indexed the same way as arrays, you can iterate through each character one at a time.

JavaScript 
let word = "LOOP"

for (let i = 0; i < word.length; i++) {
  console.log(`Index ${i}: ${word[i]}`)
}
Output

Core Concept

while Loops

Use a while loop when you do not know in advance how many iterations you will need. The loop keeps running as long as its condition is true and stops the moment it becomes false. The number of iterations is determined by what happens during the loop, not before it starts.

JavaScript 
// initialization (before the loop)
let i = 0

while (i < 5) {   // condition
  console.log(i)
  i++              // update -> without this the loop runs forever
}
Output

Unlike the for loop, the while loop separates initialization, condition, and update into different places. The counter is declared before the loop, the condition sits in the parentheses, and the update lives inside the block. Forgetting the update is one of the most common causes of infinite loops.

In the example below, the loop keeps halving a value until it drops below 1.

JavaScript 
// Keep halving a value until it drops below 1
let value = 100
let steps = 0

while (value >= 1) {
  value = value / 2
  steps++
}

console.log(`Took ${steps} steps to drop below 1`)
console.log(`Final value: ${value.toFixed(4)}`)
Output

Another common use is processing a list of items until it is exhausted. Look at the example below. The condition here is not a counter but the state of the array itself. The loop keeps going as long as there are items left, and stops the moment the array is empty.

JavaScript 
// Process items from an array until it is empty
let tasks = ["send report", "update records", "notify team"]

while (tasks.length > 0) {
  let current = tasks.pop()
  console.log(`Done: ${current}`)
}

console.log("All tasks complete")
Output
Avoid infinite loops

A while loop whose condition never becomes false will run forever and freeze the browser. Always make sure something inside the loop changes the condition such as a counter incrementing, a value shrinking, an array being emptied, so it will eventually stop.

Core Concept

for...of

for...of is a cleaner way to iterate over an array when you do not need the index. Instead of managing a counter variable, you declare a name for each element and JavaScript hands them to you one at a time.

JavaScript 
let currencies = ["USD", "EUR", "GBP", "JPY"]

for (let currency of currencies) {
  console.log(`Currency: ${currency}`)
}
Output

Compare that with the equivalent for loop. Both produce the same output, but for...of removes the need to manage the index entirely.

JavaScript 
let currencies = ["USD", "EUR", "GBP", "JPY"]

// Equivalent for loop. More setup for the same result
for (let i = 0; i < currencies.length; i++) {
  console.log(`Currency: ${currencies[i]}`)
}
Output
When to use which

Use for...of when you just need each element in turn. Use a classic for loop when you need the index.

Arrays and Loops Together

The real power of arrays comes when you combine them with loops. Instead of writing logic for each element individually, you write it once and let the loop apply it across the entire array. Three patterns come up constantly in real programs.

Finding the sum

Declare a variable to hold the running total before the loop starts, then add each element to it on every iteration. By the end of the loop the variable holds the accumulated result.

JavaScript 
let rainfall = [12, 8, 22, 5, 18, 30, 14]  // mm per day
let total = 0

for (let amount of rainfall) {
  total += amount
}

console.log(`Total rainfall: ${total}mm`)
console.log(`Daily average: ${(total / rainfall.length).toFixed(1)}mm`)
Output

Filtering into a new array

Start with an empty array. Loop over the original and use an if statement to decide whether each element belongs in the result. If it does, push it in. The new array builds up as the loop runs.

JavaScript 
let temperatures = [18, 34, 27, 41, 22, 38, 15]
let heatwaveDays  = []

for (let temp of temperatures) {
  if (temp >= 35) {
    heatwaveDays.push(temp)
  }
}

console.log(`Heatwave days: ${heatwaveDays}`)
console.log(`Count: ${heatwaveDays.length}`)
Output

Finding the maximum

Recall from Module 1 that variables declared with let can be reassigned. You can update the value a let variable holds as many times as you need throughout your program. This pattern is at the heart of solving a question like finding the maximum in an array.

Start by assuming the first element is the largest. Then loop through the rest. Each time you find a value larger than the current maximum, update the variable. The key here is that you are reassigning the same variable. It does not store every value, only the current best candidate. By the end of the loop, it holds the largest value in the array.

JavaScript 
let elevations = [412, 889, 234, 1205, 567, 988]
let highest    = elevations[0]  // start with the first value as the candidate

for (let elevation of elevations) {
  if (elevation > highest) {
    highest = elevation  // replace with the new highest
  }
}

console.log(`Highest elevation: ${highest}m`)
Output

Summary

Module Summary

Key takeaways

  • A loop runs a block of code repeatedly, letting you write logic once and apply it across a collection of values.
  • Use a for loop when the number of iterations is known before the loop starts. It manages a counter variable through three parts: start, condition, update.
  • Use a while loop when the number of iterations depends on a condition that changes during the loop. Always make sure the condition will eventually become false.
  • Use for...of when you need each element and do not need the index. It is cleaner and less error-prone for straightforward array iteration.
