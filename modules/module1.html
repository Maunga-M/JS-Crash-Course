Core Concept

What is JavaScript?

JavaScript is a high-level programming language primarily used to make web pages interactive.

It was created in 1995 by Brendan Eich while working at Netscape. It was originally built to add interactivity to web pages and it remains the only programming language that runs natively inside every web browser (which means no installation is needed to run it.).

In 2009, a tool called Node.js made it possible to run JavaScript on servers and computers outside the browser entirely. Today JavaScript is one of the most widely used programming languages in the world and is used to power interactive websites, mobile apps, server backends, and automation tools. For this course, you will write and run JavaScript directly in the browser.

Core Concept

Data Types

Before Amara's script can do anything useful, it needs to hold information: student names, exam scores, whether someone passed or failed. JavaScript needs to know what kind of information it is working with, because the rules are different for each kind. You cannot add a name and a number the same way you add two numbers. This is what data types are for.

Every value in JavaScript has a type. JavaScript divides its types into two groups: primitive types and object types.

Primitive types

Primitive types are the simplest building blocks. Each one holds a single, straightforward value. There are seven in JavaScript. You will use the first four constantly. The last three are more specialised and you may rarely encounter them as a beginner.

String
Text of any length, wrapped in quotes. You can use single quotes, double quotes, or backticks.
"Nairobi" 'Mount Kilimanjaro'
Number
Any numeric value, whether a whole number or a decimal. Integers and decimals share one type in JavaScript.
88 72.5
Boolean
Exactly two possible values. Used whenever you need a yes or no answer in your code.
true false
undefined
The only possible value is undefined. A variable that has been declared but not given a value holds this automatically.
undefined
null
The only possible value is null. Used intentionally to say "there is no value here."
null
BigInt
Integers too large for the Number type to represent accurately. Note the n at the end.
9007199254740991n
Symbol
A unique, immutable identifier. Used to avoid naming collisions in advanced code.
Symbol("id")

Object types

Object types can hold multiple pieces of data grouped together. You will not work with these until Module 7 and onwards, but it is good to know they exist. Everything in JavaScript that is not a primitive is an object type.

Object
A collection of key-value pairs written inside curly braces {}. The foundation of structured data in JavaScript.
{ product: "laptop" } { price: 49, inStock: true }
Array
An ordered list of values written inside square brackets []. Each item is accessed by its position number called an index.
[3, 7, 12, 45] ["jazz", "soul", "afrobeats"]
Function
A reusable block of code. In JavaScript, functions are also a type of object.
function greet() {}
A common mistake

"88" and 88 are not the same thing. The first is a string. The second is a number. JavaScript will treat them very differently when you try to do arithmetic. You will see exactly what happens later in this module.

Core Concept

console.log

You now know that data has types. But how do you actually see what your code does when it runs? That is what console.log() is for. It prints whatever you pass inside the parentheses. Every developer uses it constantly to check that their code is working correctly.

In this course, the output appears in the panel directly below your code. In a browser, it goes to the developer console, which you can open with Ctrl + Shift + I on Windows or Cmd + Option + I on Mac.

Below we can pass a raw value directly into console.log(). You can edit and print your own statements to the console

JavaScript 
console.log("Mount Everest")
console.log(8849)
console.log(false)
Output

You can also print multiple values at once by separating them with commas. JavaScript will print them on the same line with a space between each one.

JavaScript 
console.log("Planet:", "Mars")
console.log("Moons:", 2, "->", "Largest Moon:", "Phobos")
console.log("Has rings:", false)
Output

Core Concept

Comments

A comment is a line in your code that JavaScript ignores completely. Comments exist for you, not for the computer. You use them to explain what a section of code does, leave notes for yourself and other developers, or temporarily disable a line you are testing. You will now start seeing comments in the code examples throughout this course.

Single-line comments

A single-line comment starts with //. Everything to the right of those two slashes on the same line is ignored by JavaScript.

JavaScript 
// This entire line is a comment and will be ignored
console.log("This will print")

console.log("This will also print") // comment at end of a line

// console.log("This will NOT print because the whole line is commented out")
Output

Multi-line comments

A multi-line comment starts with /* and ends with */. Everything between those markers is ignored, no matter how many lines it spans.

JavaScript 
/*
  This is a multi-line comment.
  JavaScript ignores everything between the markers.
  Useful for longer explanations.
*/
console.log("Only this line runs")
Output

Core Concept

typeof

Earlier you saw that "88" and 88 are different types even though they look similar. typeof is the tool that tells you what type any value actually is. It returns a string describing the type. It is one of the most useful tools for debugging, especially when you are not sure why your code is behaving unexpectedly.

JavaScript 
console.log(typeof "Nairobi")  // string
console.log(typeof 88)         // number
console.log(typeof true)       // boolean
console.log(typeof "88")       // string, not a number!
console.log(typeof 88)         // number
Output

Core Concept

Variables

So far every value has been typed directly into console.log(). That works for a single print, but if Amara needs to use a student's score in several places, like calculating an average, checking if they passed, and printing a report, she would have to type the same number every time. If the score changes, she has to find and update every occurrence. Variables solve this problem. A variable is a named container that stores a value so you can reference it by name as many times as you need.

Declaration

When we declare a variable, we create a space for it in the computer's memory. We write a declaration keyword followed by a variable's name. At this point the variable exists but holds no value. Its type and value is automatically undefined.

JavaScript 
let ingredient
console.log(ingredient) // undefined
Output

There are three keywords you can use to declare a variable: let, const, and var. For this course, we will use let for everything.

let

Use let for values that may change. E.g A running total or a result that gets updated as the program runs.

const

Use const for values that should never change. If your code accidentally tries to reassign a const to a new value, JavaScript throws an error immediately.

var

var is the original keyword from early JavaScript but try to avoid using it. It has inconsistent behaviour that can cause subtle bugs. You may see it in older code but should not use it yourself.

Always use a keyword

When you introduce a new variable, always write let or const in front of it. Writing country = "France" without a keyword does not declare a variable in the normal way. It creates what is called an implicit global, which can cause unexpected bugs that are very hard to track down. Make it a habit to always declare explicitly.

Initialisation

Initialisation means giving a variable its first value. You do this with the = operator, also known as the assignment operator. This is not "equals" in the mathematical sense. It means "store this value in this variable." The value on the right is stored in the container (variable) on the left.

JavaScript 
let ingredient
ingredient = "flour"
console.log(ingredient) // flour
Output

Declaration and initialisation together

Most of the time you will do both on the same line. This is the most common form you will see in the wild.

JavaScript 
let vehicleType = "bicycle"
let wheelCount  = 2
let isElectric  = false

console.log(vehicleType)
console.log(wheelCount)
console.log(isElectric)
Output

Reassigning a variable

Once a variable has been declared, you can update its value at any time by assigning a new one. You do not write let again -- that would create a new variable. You just use the variable name on the left and the new value on the right.

JavaScript 
let score = 45
console.log(score)  // 45

score = 72
console.log(score)  // 72

score = 88
console.log(score)  // 88
Output
No duplicate names

You cannot declare two variables with the same name using let. JavaScript will throw an error. If you want to update a value, reassign it -- do not declare it again.

Core Concept

Naming Conventions

A variable name should tell you what it contains. A name like 'x' gives you no information. A name like bookTitle tells you exactly what is stored. The convention in JavaScript is camelCase. The first word starts with a lowercase letter and then the first letter of each word that follows is capitalised e.g firstName or isLoggedIn

There are also rules that JavaScript enforces. Names must start with a letter, an underscore, or a $ sign. They cannot contain spaces or special characters like & or #. They also cannot be reserved keywords.

Valid names
bookTitle
pageCount
isAvailable
_total
x valid but not descriptive
Invalid names
1stBook starts with a number
book title contains a space
return reserved keyword
book-title hyphens not allowed
book&author & is not allowed
Reserved keywords

JavaScript reserves certain words for its own use. Words like let, const, return, if, function, and for all have special meanings built into the language. So you cannot use them as variable names. If you try, JavaScript will throw an error because it reads the word as a command and not a name.

Summary

Module Summary

Key takeaways

  • JavaScript is an interpreted language that runs in the browser and beyond.
  • Every value has a type which is either primitive or an object. Primitive types include string, number, boolean, undefined, null, BigInt, and Symbol. Object types include objects, arrays, and functions.
  • console.log() prints values so you can see what your code is doing.
  • Comments let you leave notes in your code. Use // for a single line and /* */ for multiple lines.
  • typeof returns the type of any value as a string.
  • Variables are named containers for values. Use let for values that change and const for values that should not. Avoid var.
  • Variable names must start with a letter, underscore, or $. Use camelCase. Do not use reserved keywords or spaces.
