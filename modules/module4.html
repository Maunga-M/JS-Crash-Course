Core Concept

Why Types Matter

In Module 1 you learned that every value in JavaScript has a type. In Module 2 you saw that arithmetic works on numbers. In Module 3 you saw that strings have their own set of rules. These modules kept things separate. In practice, values of different types end up in the same expression all the time, and that is where things get interesting.

JavaScript is what is called a dynamically typed language. You do not declare what type a variable holds. JavaScript figures it out from the value. And when you combine two values of different types in an operation, JavaScript does not stop and ask you what you meant. It makes a decision on its own. Sometimes that decision is what you wanted. Sometimes it is not.

There are two ways types change in JavaScript. Type coercion is when JavaScript converts types implicitly and automatically, with no intervention from your end. Explicit conversion is when you do it deliberately using a conversion function. This module covers both. But first we need to understand the concept of truthy and falsy values.

3
Core Concept

Truthy and Falsy

In JavaScript, every value has an inherent boolean quality. When JavaScript needs to evaluate something as true or false, it does not require it be an actual boolean. It takes whatever value it finds and asks: is this truthy or falsy?

There are exactly six falsy values in JavaScript. Everything else is truthy.

false
The boolean value itself. Obviously falsy.
false
0
The number zero. Only zero is falsy. Every other number, positive or negative, is truthy.
0
""
An empty string. A string with even one space inside is truthy.
"" ''
null
Intentionally empty. Always falsy.
null
undefined
A variable declared but not given a value. Always falsy.
undefined
NaN
Not a Number. The result of an invalid arithmetic operation. Always falsy.
NaN

A few things worth highlighting. The number 0 is the only falsy number: -1, 0.1, and 100 are all truthy. An empty string is the only falsy string. The string "0" and a string containing just a space are truthy because they are not empty.

JavaScript 
// Falsy values
console.log(Boolean(false))      // false
console.log(Boolean(0))           // false
console.log(Boolean(""))          // false
console.log(Boolean(null))        // false
console.log(Boolean(undefined))   // false
console.log(Boolean(NaN))         // false
Output
JavaScript 
// Everything else aside from the 6 above is truthy including these
console.log(Boolean(1))          // true
console.log(Boolean(-1))         // true - any non-zero number
console.log(Boolean("0"))        // true - non-empty string
console.log(Boolean(" "))        // true - a string with a space is not empty
console.log(Boolean("false"))   // true - non-empty string
Output
The string "false" is truthy

The string "false" looks like it should be falsy, but JavaScript only cares about one thing when evaluating a string: is it empty or not? It has five characters in it, so it is truthy. The content of the string is irrelevant. Only an empty string is falsy.

4
Core Concept

Coercion

Coercion is what happens when JavaScript converts a value from one type to another automatically, without you writing any conversion code. It kicks in whenever you use an operator with values of different data types. The results follow rules, but those rules are not always obvious.

The + operator

+ is the only arithmetic operator that works on both numbers and strings. When one side is a string, JavaScript assumes you want concatenation and converts the other side to a string. This is why "5" + 2 gives you "52" instead of 7.

JavaScript 
console.log("5" + 2)      // "52" - number coerced to string
console.log(2 + "5")      // "25" - same, order does not matter
console.log("10" + "20")  // "1020" - both strings, both concatenate
console.log(10 + 20)      // 30 - both numbers, addition
Output

Every other arithmetic operator

Subtraction, multiplication, division, and remainder only work on numbers. When JavaScript sees a string in these operations, it tries to convert the string to a number first. If the string contains a valid number, the conversion works. If not, the result is NaN.

JavaScript 
// String looks like a number - JavaScript converts it
console.log("10" - 3)   // 7
console.log("6" * 2)    // 12
console.log("9" / 3)    // 3

// String does not look like a number - NaN
console.log("hello" - 1)  // NaN - hello cannot be changed to a valied number
console.log("bye" * 2)  // NaN
Output

Booleans in arithmetic

Booleans also get coerced when used in arithmetic. true becomes 1 and false becomes 0.

JavaScript 
console.log(true + true)    // 2 - true becomes 1
console.log(4 - false)   // 4 - false becomes 0
console.log(true * "12")  // 12 - both coerced to valid numbers
Output
Coercion can hide bugs

The danger of coercion is that your code does not throw an error. It just produces a wrong result silently. A number being treated as a string, or a string being converted to NaN, can cause values to be wrong many lines later with no obvious indication of where things went wrong. This is why explicit conversion, which you are in control of, is safer whenever types might be mixed.

Core Concept

Explicit Conversion

Explicit conversion means you are in charge. Instead of letting JavaScript decide what to do with mismatched types, you call a function that converts the value to exactly the type you need. There are three built-in conversion functions we will look at.

Number()

Number() attempts to convert any value to a number. Strings that contain a valid number convert cleanly. Strings that contain anything else return NaN. true converts to 1 and false converts to 0. null converts to 0. undefined converts to NaN.

JavaScript 
console.log(Number("42"))       // 42
console.log(Number("3.14"))     // 3.14
console.log(Number(""))          // 0 - empty string becomes 0
console.log(Number("hello"))    // NaN - not a valid number
console.log(Number(true))       // 1
console.log(Number(false))      // 0
console.log(Number(null))       // 0
console.log(Number(undefined))  // NaN
Output

String()

String() converts any value to its text representation. It almost never fails. Every value has a string form, even null, undefined, and NaN.

JavaScript 
console.log(String(42))         // "42"
console.log(String(3.14))       // "3.14"
console.log(String(true))       // "true"
console.log(String(false))      // "false"
console.log(String(null))       // "null"
console.log(String(undefined))  // "undefined"
console.log(String(NaN))        // "NaN"
Output

Boolean()

Boolean() converts any value to true or false. The six falsy values return false, everything else returns true. It does not matter what the value is or what it means. Boolean() only asks one question for each type:

For numbers

Is it zero? Zero and negative zero are the only falsy numbers. Every other number, no matter how large or small, positive or negative, is truthy. Boolean() does not care about the value itself. It only cares whether it is zero or not.

For strings

Is it empty? An empty string "" is falsy. A string with any content at all, even just a space, is truthy. The string "false" is truthy because it is not empty.

For everything else

null, undefined, and NaN are always falsy. Objects, arrays, and functions are always truthy, even an empty object {} or empty array [].

JavaScript 
// Numbers - only zero is falsy
console.log(Boolean(0))      // false
console.log(Boolean(100))    // true
console.log(Boolean(-99))    // true
console.log(Boolean(0.001))  // true

// Strings - only empty is falsy
console.log(Boolean(""))       // false
console.log(Boolean("hello"))  // true
console.log(Boolean("0"))      // true - not empty
console.log(Boolean("false"))  // true - not empty
Output

Core Concept

parseInt() and parseFloat()

Number() is strict. If the string contains anything that is not a valid number, it returns NaN. That is sometimes too strict. In the real world, strings often have numbers mixed with other characters: a price like "24px", a measurement like "180cm", or user input like "42 years old". parseInt() and parseFloat() are more forgiving. They read from the beginning of the string and extract the number they find, stopping when they hit a character that is not part of a number.

JavaScript 
// Number() is strict
console.log(Number("42px"))     // NaN
console.log(Number("180cm"))    // NaN

// parseInt reads the leading integer and stops
console.log(parseInt("42px"))   // 42
console.log(parseInt("180cm"))  // 180

// parseFloat reads the leading decimal and stops
console.log(parseFloat("3.14rem"))   // 3.14
console.log(parseFloat("9.81m/s"))   // 9.81
Output

The key difference between the two is that parseInt() strips the decimal part and returns a whole number. parseFloat() keeps it. If the string does not start with a number, both return NaN.

JavaScript 
// parseInt strips decimals
console.log(parseInt("3.99"))    // 3 - not rounded, just truncated
console.log(parseFloat("3.99"))  // 3.99

// If the string does not start with a number, both return NaN
console.log(parseInt("hello"))    // NaN
console.log(parseInt("px42"))     // NaN - number must come first
Output
When to use which

Use Number() when you expect a clean numeric string and want to catch anything unexpected with NaN. Use parseInt() when you know the string contains a whole number possibly followed by other characters. Use parseFloat() when the number may have a decimal part. These are not interchangeable choices. Just pick the one that matches what your data actually looks like.

Module Summary

Key takeaways

  • There are exactly six falsy values in JavaScript: false, 0, "", null, undefined, and NaN. Everything else is truthy.
  • For numbers, only zero is falsy. For strings, only an empty string is falsy.
  • Coercion happens automatically. With +, a string on either side triggers concatenation. With any other operation, JavaScript tries to convert strings to numbers.
  • Number() converts cleanly but returns NaN for anything that does not look like a number.
  • String() converts any value to its text form.
  • Boolean() converts based on truthiness. It does not care about the value itself, only whether it is one of the six falsy values.
  • parseInt() extracts a leading whole number from a string. parseFloat() extracts a leading decimal. Both return NaN if no number is found at the start.
