Core Concept

What is a Conditional?

In Module 5 you learned how to compare values and combine conditions. But a comparison on its own does not do anything. It just produces a result and that result is not useful unless something acts on it. We can use conditionals to make that result useful in our code. It is a structure that says: if this condition is true, run this code; otherwise, do something else or nothing at all.

Conditional statements are what turn a passive script into a program that dynamically responds to its data. Without them, every line of code runs every time, in the same order, regardless of what the values actually are. With conditionals, different paths through the code become possible. A price might be calculated one way for members and another way for guests. Or an error message might appear only when input is missing. A result might be labelled differently depending on how high a value is. All of this requires conditionals.

3
Core Concept

The if Statement

The if statement is the most fundamental conditional. It starts by taking a condition in parentheses. If the condition evaluates to something truthy, the block of code inside the curly braces runs. If the condition is falsy, the block is skipped entirely.

JavaScript 
let balance = 200

if (balance > 0) {
  console.log("Funds available")
}

console.log("Done")
Output

Change the balance to 0 and run it again. The message disappears but "Done" still prints. The code after the block always runs but the code inside the braces is conditional.

Always use curly braces

JavaScript allows you to omit the curly braces when the block contains only one line. As a beginner, this may be a common source of bugs. Always include the braces for now. They make it explicit which lines belong to the conditional and prevent mistakes when you add a second line later.

4
Core Concept

Truthy Conditions

An if statement does not require a boolean. JavaScript automatically converts whatever you put in the condition to either true or false. This means you can write conditions that are shorter and more direct but only when it is appropriate to do so.

When shorthand is appropriate

Use shorthand when you only care whether something exists or is present, not what the specific value is.

JavaScript 
let isSunny  = true
let userName = "Kofi"

// Is it true?
if (isSunny) {
  console.log("It is sunny")
}

// Is there a value?
if (userName) {
  console.log("Hello, " + userName)
}

// Writing === true is unnecessary here because JavaScript already converts it to a boolean
if (isSunny === true) {
  console.log("This works but is redundant")
}
Output

When you must use explicit comparison

Use an explicit comparison when you care about a specific value, not just whether something is present.

JavaScript 
let count = 0

// We want to check the that count is exactly zero

// WRONG -> 0 is falsy, so this block never runs even though count exists
if (count) {
  console.log("This will not run")
}

// CORRECT -> === 0 checks the exact value, not just whether count has a value
if (count === 0) {
  console.log("This will run")
}
Output
Simple rule

Use if (value) when you mean "does this exist or is it set?" Use if (value === x) when you mean "does this equal exactly this?" The shorthand is a convenience, not a replacement for all comparisons.

5
Core Concept

else

An if statement runs its block when the condition is true. But what happens when it is false and we want an alternative output? The else block handles that. If the if condition evaluates to false, the if block is skipped and the else block runs instead. Exactly one of the two will always run but not both.

JavaScript 
let stock = 2

if (stock > 10) {
  console.log("Plenty of stock")
} else {
  console.log("Low stock")
}
Output

The else branch runs because 2 is not greater than 10, so the condition is false. Change the stock to 50 and run again. Now the condition is true and the if branch executes instead of the else.

6
Core Concept

else if

When there are more than two possible outcomes, you can chain conditions using else if. JavaScript evaluates each condition from top to bottom and runs the first block whose condition is true. Once a match is found, the remaining branches are skipped entirely, even if they would also be true.

JavaScript 
let duration = 95

if (duration >= 120) {
  console.log("Long flight")
} else if (duration >= 60) {
  console.log("Medium flight")
} else if (duration >= 30) {
  console.log("Short flight")
} else {
  console.log("Very short flight")
}
Output

Order matters

JavaScript stops at the first condition that matches. If you put a less restrictive condition above a more restrictive one, the more restrictive branch will never run. Here is the same logic written in the wrong order.

JavaScript 
let duration = 95

// Wrong order -> the greater than or equal to 30 check matches first and always executes
if (duration >= 30) {
  console.log("Short flight")
} else if (duration >= 60) {
  console.log("Medium flight")
} else if (duration >= 120) {
  console.log("Long flight")
}
Output

A duration of 95 should print "Medium flight" but the wrong order makes it print "Short flight" instead. Neither of the two else if branches will ever be reached. Therefore, always start with the most restrictive condition and work downward.

You must always start with an if statement if using conditionals. You can have as many else if branches as you need. The final else is optional but acts as a safety net for any value that does not match any condition above it.

Core Concept

Nested Conditionals

You can place an if statement inside another if block. This is called nesting. But nesting adds indentation and makes code harder to follow. Every extra level of nesting is another thing a reader has to hold in their head while parsing the logic.

Here is a nested example that checks whether a user can access a feature.

JavaScript 
let hasTicket  = true
let isValidAge = true

// Nested conditional
if (hasTicket) {
  if (isValidAge) {
    console.log("Entry granted")
  } else {
    console.log("Age restriction applies")
  }
} else {
  console.log("No ticket")
}
Output

The same logic can be expressed with logical operators in a single level. This is easier to read because the conditions are stated together rather than buried inside each other.

JavaScript 
let hasTicket  = true
let isValidAge = true

// Flattened -> same result but easier to read
if (hasTicket && isValidAge) {
  console.log("Entry granted")
} else if (hasTicket && !isValidAge) {
  console.log("Age restriction applies")
} else {
  console.log("No ticket")
}
Output
Prefer flat over nested

If you find yourself writing a third level of nesting, stop and ask whether the conditions can be combined with && or ||. Flat code is almost always easier to read, test, and change later.

Core Concept

Ternary Operator

The ternary operator is a compact way to write a two-branch conditional in a single expression. It takes a condition, a value to use if the condition is true, and a value to use if it is false. The syntax is
condition ? valueIfTrue : valueIfFalse.

It is most useful when you need to assign one of two values to a variable based on a condition. For anything more complex than that, a regular if/else is clearer.

JavaScript 
let fileSize = 2.4

// if/else version
let sizeLabel
if (fileSize > 1) {
  sizeLabel = "Large"
} else {
  sizeLabel = "Small"
}
console.log(sizeLabel)

// Ternary version -- same result
let sizeLabel2 = fileSize > 1 ? "Large" : "Small"
console.log(sizeLabel2)
Output

Because the ternary is an expression that returns a value, it can be placed directly inside ${} in a template string.

JavaScript 
let downloadSpeed = 85

// Ternary embedded directly inside a template string
console.log(`Connection: ${downloadSpeed > 50 ? "Fast" : "Slow"}`)

let remaining = 3
console.log(`Items left: ${remaining} (${remaining === 1 ? "item" : "items"})`)
Output
Do not nest ternaries

You can chain ternaries to handle more than two outcomes, but the result becomes difficult to read very quickly. If you need more than two branches, use if/else if instead.

Summary

Module Summary

Key takeaways

  • An if statement runs a block of code only when its condition is true. The condition can be any expression, not just a boolean.
  • else handles the default case when no if condition is met.
  • else if is used to handle multiple outcomes. JavaScript starts at the if statements, executes at the first match and skips the rest. Order matters.
  • The ternary operator is a compact two-branch conditional. Use it for simple value assignments. Use if/else for anything more complex.
